
# Playwright Revision Notes

## Topic 1 — Browser Launch

| #  | Concept               | One-liner                                                                                                                 | Syntax                                                           | Example                                                        |
| -- | --------------------- | ------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------- | -------------------------------------------------------------- |
| 1  | Engines               | Playwright ships 3 engines — chromium, firefox, webkit — each imported separately                                       | `import { chromium, firefox, webkit } from '@playwright/test'` | `const browser = await chromium.launch();`                   |
| 2  | `.launch()`         | Starts a new browser**process**                                                                                     | `engine.launch(options)`                                       | `await chromium.launch({ headless: false });`                |
| 3  | `headless`          | Show (`false`) or hide (`true`, default) the browser UI                                                               | `{ headless: boolean }`                                        | `await chromium.launch({ headless: false });`                |
| 4  | `args`              | Pass engine-specific command-line flags (flag format differs per engine)                                                  | `{ args: string[] }`                                           | `await chromium.launch({ args: ['--start-maximized'] });`    |
| 5  | `slowMo`            | Adds a ms delay before every action — for debugging/demos, not CI                                                        | `{ slowMo: number }`                                           | `await chromium.launch({ slowMo: 100 });`                    |
| 6  | `timeout`           | Max ms to wait for the**browser process** to start (not navigation)                                                 | `{ timeout: number }`                                          | `await chromium.launch({ timeout: 60000 });`                 |
| 7  | `channel`           | Launches the**real, system-installed** browser instead of Playwright's bundled build                                | `{ channel: 'chrome' \| 'msedge' \| 'firefox' }`                 | `await chromium.launch({ channel: 'chrome' });`              |
| 8  | No channel for webkit | webkit has no`channel` option — always uses Playwright's own build                                                     | —                                                               | `await webkit.launch({ headless: false });`                  |
| 9  | Edge                  | Edge is Chromium-based, so use the`chromium` engine with `channel: 'msedge'`                                          | `chromium.launch({ channel: 'msedge' })`                       | `const edge = await chromium.launch({ channel: 'msedge' });` |
| 10 | Bundled vs installed  | From v1.57+, default chromium build is "Chrome-to-Test" — a Google-maintained testing binary, no channel needed for that | —                                                               | `chromium.launch()`                                          |

**🎯 Interview point:** Use `channel` when a bug is browser-version-specific and you need to reproduce it against the exact Chrome/Edge/Firefox build your real users have installed.

## Topic 3 — Navigation Commands

| #  | Concept                           | One-liner                                                                         | Syntax                                | Example                                                                 |
| -- | --------------------------------- | --------------------------------------------------------------------------------- | ------------------------------------- | ----------------------------------------------------------------------- |
| 1  | `goto()`                        | Navigates the page and returns a Promise                                          | `page.goto(url, options?)`          | `await page.goto(URL, { timeout: 40000, waitUntil: 'networkidle' });` |
| 2  | `waitUntil: 'commit'`           | Done as soon as the first byte is received — fastest, least reliable             | `{ waitUntil: 'commit' }`           | `await page.goto(URL, { waitUntil: 'commit' });`                      |
| 3  | `waitUntil: 'domcontentloaded'` | HTML parsed, DOM ready — images may still load; good speed/reliability balance   | `{ waitUntil: 'domcontentloaded' }` | `await page.goto(URL, { waitUntil: 'domcontentloaded' });`            |
| 4  | `waitUntil: 'load'`             | Default — waits for entire page (images, scripts, everything) to finish          | `{ waitUntil: 'load' }`             | `await page.goto(URL);`                                               |
| 5  | `waitUntil: 'networkidle'`      | Waits until 0 network requests for 500ms — best for SPAs with async data fetches | `{ waitUntil: 'networkidle' }`      | `await page.goto(URL, { waitUntil: 'networkidle' });`                 |
| 6  | `goBack()`                      | Navigates to the previous page in session history                                 | `page.goBack(options?)`             | `await page.goBack({ waitUntil: 'load', timeout: 5000 });`            |
| 7  | `goForward()`                   | Navigates forward — only works if a`goBack()` happened first                   | `page.goForward(options?)`          | `await page.goForward({ waitUntil: 'load' });`                        |
| 8  | `reload()`                      | Refreshes the current page — used for "session expired" / data-refresh checks    | `page.reload(options?)`             | `await page.reload({ waitUntil: 'load' });`                           |
| 9  | `url()`                         | Sync — returns the current URL string, no`await` needed                        | `page.url()`                        | `const currentURL = page.url();`                                      |
| 10 | `title()`                       | Async — reads the`<title>` tag text                                            | `await page.title()`                | `const pageTitle = await page.title();`                               |
| 11 | `content()`                     | Async — returns the full rendered HTML as a string                               | `await page.content()`              | `const html = await page.content();`                                  |

**🎯 Interview point:** Most teams default to `domcontentloaded` for speed, but switch to `networkidle` for pages that fetch data via AJAX/fetch **after** the initial HTML loads — otherwise the test may interact before dynamic content appears.
