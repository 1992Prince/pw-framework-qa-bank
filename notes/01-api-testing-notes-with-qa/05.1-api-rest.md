
# Types of APIs & Web Services

> **Interview Tip 💡**
>
> Interviewers usually ask:
>
> - What are APIs?

> - API vs Web Service?
> - Types of Web Services?
>
> Don't memorize definitions. Remember **who can access it** and **how it communicates**.

---

# What is an API?

> **API (Application Programming Interface)** is a medium that allows two applications to communicate with each other.

Instead of directly accessing the database, applications communicate through APIs.

### Airtel Example

```text
Customer
    │
Airtel App
    │
GET /offers
    │
Offer Service
    │
Database
    │
Offer Response
```

The mobile app never talks directly to the database.

It always goes through APIs.

---

# Types of APIs (Based on Access)

## 1. Open APIs (Public APIs)

> APIs that are available for anyone to use.

Anyone can access them after following basic documentation or authentication.

### Examples

- Google Maps API
- Weather API
- Currency Exchange API

### Airtel Example

Suppose Airtel exposes an API to check network coverage.

```http
GET /networkCoverage?city=Delhi
```

Anyone can use it.

---

## 2. Partner APIs

> APIs shared only with trusted business partners.

Access is restricted using API Keys, OAuth, etc.

### Airtel Example

Airtel partners with Amazon.

Amazon calls Airtel APIs to verify recharge status.

```text
Amazon
     │
Partner API
     │
Airtel Recharge Service
```

Only Amazon can access it.

---

## 3. Private APIs (Enterprise APIs) ⭐

> APIs used only inside the organization.

These APIs are not exposed to external users.

### Airtel Example

Customer logs in.

```text
Login Service
      │
Customer Service
      │
Offer Service
      │
Billing Service
```

These APIs are internal.

As SDETs, **most of the APIs we test are Private APIs.**

# What is a Web Service?

> A **Web Service** is an API that communicates over the internet using web protocols like **HTTP or HTTPS**.

It enables communication between two applications over a network.

### Simple Definition

> **Every Web Service is an API, but not every API is a Web Service.**

---

# Airtel Example

```text
Airtel App
      │
HTTPS
      │
Recharge Service
```

Since communication happens over HTTP/HTTPS,

this is a **Web Service**.

---

# API vs Web Service

| API                                       | Web Service                                     |
| ----------------------------------------- | ----------------------------------------------- |
| Allows communication between applications | A type of API that communicates over HTTP/HTTPS |
| May or may not use the internet           | Uses web protocols                              |
| Can use different communication methods   | Uses HTTP/HTTPS                                 |
| Can be local or remote                    | Usually remote                                  |
| Broader concept                           | Subset of APIs                                  |

# Types of Web Services

Modern projects mainly use:

- REST APIs ⭐⭐⭐
- SOAP APIs
- GraphQL APIs
- WebSockets
- RPC (Remote Procedure Call)

---

# 1. REST APIs ⭐ (Most Common)

> REST is an architectural style that uses HTTP methods and JSON for communication.

Uses

- GET
- POST
- PUT
- DELETE
- PATCH

Returns

```json
{
   "name":"Prince"
}
```

### Airtel Example

```http
GET /offers
```

Returns

```json
[
   "₹299",
   "₹399"
]
```

### Why Popular?

✔ Simple

✔ Fast

✔ JSON

✔ Lightweight

✔ Easy to automate

---

# 2. SOAP APIs

> SOAP (Simple Object Access Protocol) is a protocol for exchanging structured XML messages.

Uses

- XML
- WSDL
- Strict Standards

Example

```xml
<Customer>
   <Name>Prince</Name>
</Customer>
```

### Where Used?

- Banking
- Insurance
- Government
- Legacy Enterprise Systems

---

# REST vs SOAP

| REST                     | SOAP                      |
| ------------------------ | ------------------------- |
| Uses JSON                | Uses XML                  |
| Lightweight              | Heavy                     |
| Faster                   | Slower                    |
| Easy                     | Complex                   |
| Most modern applications | Legacy enterprise systems |

---

# 3. GraphQL APIs

> GraphQL allows clients to request only the data they need.

Instead of multiple REST calls,

one GraphQL query can return everything.

### REST Example

```text
GET /customer

GET /offers

GET /billing
```

Three API calls.

---

### GraphQL

```graphql
{
 customer{
   name
   offers
   billing
 }
}
```

Single request.

---

### Airtel Example

Customer Dashboard

Need

- Profile
- Recharge
- Offers
- Bills

GraphQL returns everything in one request.

---

# 4. WebSockets

> WebSockets provide two-way real-time communication between client and server.

Unlike HTTP,

the connection stays open.

### Examples

- Live Chat
- WhatsApp
- Stock Market
- Cricket Scores

### Airtel Example

Customer Support Chat.

```text
Customer
    ⇄
Support Agent
```

Messages appear instantly.

---

# HTTP vs WebSocket

| HTTP                         | WebSocket                 |
| ---------------------------- | ------------------------- |
| Request → Response          | Two-way communication     |
| New connection every request | One persistent connection |
| Normal APIs                  | Real-time applications    |

---

# 5. RPC (Remote Procedure Call)

> RPC allows one application to execute a function on another server as if it were a local method.

Instead of calling URLs,

you call methods.

Example

```text
calculateBill()
```

instead of

```http
POST /billing
```

Popular RPC Frameworks

- gRPC
- Apache Thrift

---

### Airtel Example

Billing Service

calls

```text
calculateRechargeAmount()
```

instead of multiple REST endpoints.

# Interview Answer (1 Minute)

> APIs allow different applications to communicate with each other. Based on access, APIs are of three types: Open APIs for public use, Partner APIs for trusted business partners, and Private APIs used internally within an organization. In my telecom project, most APIs are Private APIs because they are consumed only by internal services.
>
> A Web Service is a type of API that communicates over web protocols like HTTP or HTTPS. Modern applications mainly use REST APIs because they are lightweight and use JSON. SOAP is used in many legacy enterprise systems, GraphQL allows clients to fetch only the required data, WebSockets are used for real-time communication like chat, and RPC/gRPC is commonly used for high-performance service-to-service communication.
