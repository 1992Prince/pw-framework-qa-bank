# Playwright Hooks — Complete Notes

## Library Architecture — How Playwright Exports Hooks

### Step 1 — Playwright Library Se test Object Import Karte Ho

```javascript
import { test, expect } from "@playwright/test";
```

Yeh line mein:
- `test` — ek object import ho raha hai jo Playwright ne export kiya hai
- `expect` — ek function import ho raha hai

---

### Step 2 — test Object Ka Structure

Playwright ne internally test object aise banaya hai:

```javascript
// Playwright ke library mein (internally)
export const test = {
  // Test define karne ke liye
  (name, callback) => { /* test implementation */ },
  
  // Hooks attach karte hain
  beforeAll: (callback) => { /* beforeAll implementation */ },
  afterAll: (callback) => { /* afterAll implementation */ },
  beforeEach: (callback) => { /* beforeEach implementation */ },
  afterEach: (callback) => { /* afterEach implementation */ },
  
  // Aur bhi methods hote hain
  describe: (name, callback) => { /* describe implementation */ },
  only: (...) => { /* only implementation */ },
  skip: (...) => { /* skip implementation */ },
  
  // ... aur bhi
};
```

Toh `test` ek object hai jisme multiple methods attach hain.

---

### So To Access Hooks of Playwright We Need To Use test.beforeAll, test.afterAll, ...

Hooks ko access karne ke liye humne `test.` prefix use karna padta hai.

---

### Agar Directly beforeAll Use Karte Toh?

```javascript
// ❌ GALAT

beforeAll(async () => {  // beforeAll globally available nahi hai
  console.log("Setup");
});

test("Test 1", async ({ page }) => {
  // test code
});
```

**Error:**
```
ReferenceError: beforeAll is not defined
```

`beforeAll` directly available nahi hai kyunki yeh Playwright ke `test` object ke andar define hai.

---

### Methods Available in test Object

Aur `test` object ke andar sirf hooks nahi, bilkul lots of methods available hain:

```javascript
test                      // Direct test define karna
test.afterAll             // After all tests hook
test.afterEach            // After each test hook
test.beforeAll            // Before all tests hook
test.beforeEach           // Before each test hook
test.describe             // Group tests
test.describe.configure   // Configure test groups
test.describe.fixme       // Mark group as fixme
test.describe.only        // Run only this group
test.describe.skip        // Skip this group
test.extend               // Extend fixtures
test.fail                 // Mark test as expected to fail
test.fail.only            // Run only this failing test
test.fixme                // Mark test as fixme (will fix later)
test.info                 // Get test info
test.only                 // Run only this test
test.setTimeout           // Set timeout for test
test.skip                 // Skip this test
test.slow                 // Mark test as slow
test.step                 // Create test steps
test.step.skip            // Skip step
test.use                  // Use fixtures/configuration
```

## Part 1 — Hooks Basics

### What Are Hooks?

Hooks are special functions in Playwright that run automatically at specific points in the test lifecycle — before tests start, after tests end, before each test, after each test.

Think of hooks like ceremony before a cricket match — national anthem bajta hai (`beforeAll`), phir har batsman se pehle warm-up (`beforeEach`), khel khatam hone ke baad har batsman apna gear rakhta hai (`afterEach`), aur match khatam hone ke baad stadium saaf hota hai (`afterAll`).

**Real-Time Scenario:**
- Database connection pehle se open karo (`beforeAll`)
- Har test se pehle login karo (`beforeEach`)
- Har test ke baad data clear karo (`afterEach`)
- Database connection close karo last mein (`afterAll`)

---

### 4 Types of Hooks

Playwright mein 4 hooks hote hain:

| Hook | Runs | Frequency | Scope |
|------|------|-----------|-------|
| `beforeAll` | Before all tests start | Once | Per worker, per spec file |
| `afterAll` | After all tests end | Once | Per worker, per spec file |
| `beforeEach` | Before every single test | Per test | Per worker, per spec file |
| `afterEach` | After every single test | Per test | Per worker, per spec file |

---

## Hook 1 — test.beforeAll()

### Concept

`beforeAll()` ek baar execute hota hai — jab spec file ke test chalna shuru hote hain. Matlab agar 10 tests hain, `beforeAll` sirf ek baar run hoga — sab tests se pehle.

**Scope Important:** Agar project mein 5 spec files hain, har spec file ka apna `beforeAll` hoga. Ek spec file ka `beforeAll` doosre spec file ke tests se pehle nahi chalega. Har spec file isolated hota hai.

### Syntax

```javascript
test.beforeAll(async () => {
  // code jo ek baar execute hona hai
});
```

Ya title dena hai toh:

```javascript
test.beforeAll("Setup database connection", async () => {
  // code
});
```

> **Note:** Title optional hai. `beforeAll` bas function bhi le sakta hai, ya title + function dono.

---

### Example 1 — Simple Setup

```javascript
import { test, expect } from "@playwright/test";

test.beforeAll(async () => {
  console.log("🟢 beforeAll — Test suite starting!");
  console.log("Opening database connection...");
  // Database open karo
  // Server start karo
  // API token fetch karo
});

test("User login test 1", async ({ page }) => {
  console.log("Running test 1");
  await page.goto("https://example.com/login");
  // test code
});

test("User login test 2", async ({ page }) => {
  console.log("Running test 2");
  // test code
});
```

**Execution Order:**
```
1. beforeAll — "🟢 beforeAll — Test suite starting!"
2. beforeAll — "Opening database connection..."
3. Test 1 — "Running test 1"
4. Test 2 — "Running test 2"
```

`beforeAll` sirf ek baar hai!

---

### Example 2 — Real-Time: Database Setup

```javascript
import { test, expect } from "@playwright/test";

let dbConnection;

test.beforeAll(async () => {
  // Database connection banao
  dbConnection = await connectToDatabase({
    host: "localhost",
    port: 5432,
    database: "test_db",
    user: "admin",
    password: "secret"
  });
  
  console.log("✅ Database connected");
});

test("Test 1 - Insert user", async () => {
  const result = await dbConnection.query("INSERT INTO users (name) VALUES ('Sarika')");
  expect(result.rowCount).toBe(1);
});

test("Test 2 - Fetch user", async () => {
  const result = await dbConnection.query("SELECT * FROM users WHERE name = 'Sarika'");
  expect(result.rows.length).toBe(1);
});
```

**Kya hota hai:**
- `beforeAll` — ek baar database connect hota hai
- Test 1 — insert karta hai (same connection use karta hai)
- Test 2 — fetch karta hai (same connection)

Agar `beforeAll` nahi hota toh dono tests mein database connection banani padti — waste of time aur resources.

---

### Example 3 — Real-Time: API Token Setup

```javascript
import { test, expect } from "@playwright/test";

let apiToken;

test.beforeAll(async ({ request }) => {
  // API se token fetch karo ek baar
  const response = await request.post("https://api.example.com/auth/login", {
    data: {
      username: "testuser",
      password: "testpass123"
    }
  });
  
  const data = await response.json();
  apiToken = data.token;
  
  console.log("✅ API Token received:", apiToken);
});

test("Test 1 - Fetch user profile", async ({ request }) => {
  const response = await request.get("https://api.example.com/user/profile", {
    headers: { Authorization: `Bearer ${apiToken}` }
  });
  expect(response.status()).toBe(200);
});

test("Test 2 - Update profile", async ({ request }) => {
  const response = await request.put("https://api.example.com/user/profile", {
    headers: { Authorization: `Bearer ${apiToken}` },
    data: { name: "Updated Name" }
  });
  expect(response.status()).toBe(200);
});
```

**Benefit:** Token sirf ek baar fetch hota hai, dono tests use karte hain.

## Hook 2 — test.afterAll()

### Concept

`afterAll()` ek baar execute hota hai — jab sare tests khatam ho jayen. Iska kaam cleanup karna hai — jo resources `beforeAll` mein khole the unhe close karna.

**Analogy:** Kitchen mein khana banate ho (`beforeAll`), sab ne khana khaya (tests), aur ab bartan dhoona padta hai (`afterAll`).

### Syntax

```javascript
test.afterAll(async () => {
  // cleanup code
});
```

---

### Example 1 — Simple Cleanup

```javascript
import { test, expect } from "@playwright/test";

let dbConnection;

test.beforeAll(async () => {
  dbConnection = await connectToDatabase(...);
  console.log("✅ Database connected");
});

test("Test 1", async () => {
  // database use karo
});

test("Test 2", async () => {
  // database use karo
});

test.afterAll(async () => {
  // Database connection close karo
  await dbConnection.close();
  console.log("🔴 Database disconnected");
});
```

**Execution Order:**
```
1. beforeAll — "✅ Database connected"
2. Test 1 — (use database)
3. Test 2 — (use database)
4. afterAll — "🔴 Database disconnected"
```

---

### Example 2 — Real-Time: Cleanup Files

```javascript
import { test, expect } from "@playwright/test";
import fs from "fs";

test.beforeAll(async () => {
  // Test data files banao
  fs.writeFileSync("test-data.json", JSON.stringify({ users: [] }));
  console.log("✅ Test data file created");
});

test("Test 1", async () => {
  // file use karo
});

test("Test 2", async () => {
  // file use karo
});

test.afterAll(async () => {
  // Test files delete karo
  fs.unlinkSync("test-data.json");
  console.log("🔴 Test data file deleted");
});
```

---

### Example 3 — Real-Time: Server Cleanup

```javascript
import { test, expect } from "@playwright/test";

let server;

test.beforeAll(async () => {
  // Local test server start karo
  server = await startTestServer({
    port: 3000,
    database: "test_db"
  });
  console.log("✅ Test server started on port 3000");
});

test("API Test 1", async ({ request }) => {
  const response = await request.get("http://localhost:3000/api/users");
  expect(response.status()).toBe(200);
});

test("API Test 2", async ({ request }) => {
  const response = await request.get("http://localhost:3000/api/products");
  expect(response.status()).toBe(200);
});

test.afterAll(async () => {
  // Server stop karo aur cleanup karo
  await server.stop();
  console.log("🔴 Test server stopped");
});
```

## Hook 3 — test.beforeEach()

### Concept

`beforeEach()` har test se pehle execute hota hai. Agar 10 tests hain toh `beforeEach` 10 baar chalega — har test se pehle.

**Difference from beforeAll:**
- `beforeAll` → ek baar
- `beforeEach` → har test se pehle

### Syntax

```javascript
test.beforeEach(async ({ page }) => {
  // code jo har test se pehle run hona hai
});
```

---

### Example 1 — Login Before Every Test

```javascript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  // Har test se pehle login karo
  await page.goto("https://example.com/login");
  await page.fill('input[name="email"]', "user@example.com");
  await page.fill('input[name="password"]', "password123");
  await page.click("button:has-text('Login')");
  await page.waitForURL("**/dashboard");
  
  console.log("✅ User logged in");
});

test("Test 1 - View profile", async ({ page }) => {
  // Already logged in from beforeEach
  await page.click("text=Profile");
  expect(await page.isVisible("text=John Doe")).toBeTruthy();
});

test("Test 2 - Update settings", async ({ page }) => {
  // Already logged in from beforeEach
  await page.click("text=Settings");
  expect(await page.isVisible("text=Account Settings")).toBeTruthy();
});

test("Test 3 - Logout", async ({ page }) => {
  // Already logged in from beforeEach
  await page.click("text=Logout");
  expect(await page.url()).toContain("login");
});
```

**Execution Order:**
```
1. beforeEach — Login user
2. Test 1 — View profile
3. beforeEach — Login user (again)
4. Test 2 — Update settings
5. beforeEach — Login user (again)
6. Test 3 — Logout
```

Har test fresh start karta hai — already logged in state se.

---

### Example 2 — Reset App State Before Each Test

```javascript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  // Har test se pehle app ko fresh state mein lao
  await page.goto("https://example.com");
  
  // Clear local storage
  await page.evaluate(() => localStorage.clear());
  
  // Clear session storage
  await page.evaluate(() => sessionStorage.clear());
  
  // Clear cookies
  await page.context().clearCookies();
  
  console.log("✅ App state reset");
});

test("Test 1 - Fresh install flow", async ({ page }) => {
  // Fresh state se start
  await page.goto("https://example.com/install");
  // test code
});

test("Test 2 - Another fresh flow", async ({ page }) => {
  // Wapas fresh state se start
  await page.goto("https://example.com/onboarding");
  // test code
});
```

**Benefit:** Har test isolated hota hai — ek test ka data doosre test ko affect nahi karta.

---

### Example 3 — Create Test Data Before Each Test

```javascript
import { test, expect } from "@playwright/test";

let testUserId;

test.beforeEach(async ({ request }) => {
  // Har test ke liye naya user banao
  const response = await request.post("https://api.example.com/users", {
    data: {
      name: "Test User",
      email: `user-${Date.now()}@example.com`,
      password: "testpass123"
    }
  });
  
  const data = await response.json();
  testUserId = data.id;
  
  console.log(`✅ Test user created: ${testUserId}`);
});

test("Test 1 - User profile", async ({ request }) => {
  // Use testUserId banaya gaya beforeEach se
  const response = await request.get(`https://api.example.com/users/${testUserId}`);
  expect(response.status()).toBe(200);
});

test("Test 2 - User settings", async ({ request }) => {
  // Naya testUserId milega (beforeEach phir se chalega)
  const response = await request.get(`https://api.example.com/users/${testUserId}/settings`);
  expect(response.status()).toBe(200);
});
```

Har test ko independent user milta hai.

## Hook 4 — test.afterEach()

### Concept

`afterEach()` har test ke baad execute hota hai. Agar 10 tests hain toh `afterEach` 10 baar chalega — har test ke baad.

**Use Case:** Har test ke baad cleanup karna — database se test data delete karna, screenshots capture karna (agar fail hua), logs save karna, etc.

### Syntax

```javascript
test.afterEach(async ({ page }) => {
  // code jo har test ke baad run hona hai
});
```

---

### Example 1 — Take Screenshot on Failure

```javascript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("https://example.com");
});

test("Test 1 - Login", async ({ page }) => {
  await page.fill('input[name="email"]', "user@example.com");
  await page.fill('input[name="password"]', "password123");
  await page.click("button:has-text('Login')");
  expect(await page.isVisible("text=Dashboard")).toBeTruthy();
});

test("Test 2 - Invalid login", async ({ page }) => {
  await page.fill('input[name="email"]', "wrong@example.com");
  await page.fill('input[name="password"]', "wrongpass");
  await page.click("button:has-text('Login')");
  expect(await page.isVisible("text=Invalid credentials")).toBeTruthy();
});

test.afterEach(async ({ page }, testInfo) => {
  // Agar test fail hua toh screenshot lao
  if (testInfo.status !== 'passed') {
    const screenshotPath = `./screenshots/${testInfo.title}-failed.png`;
    await page.screenshot({ path: screenshotPath });
    console.log(`📸 Screenshot saved: ${screenshotPath}`);
  }
});
```

**Execution:**
```
1. beforeEach — goto website
2. Test 1 — Login
3. afterEach — Check if passed (yes, no screenshot)
4. beforeEach — goto website
5. Test 2 — Invalid login
6. afterEach — Check if passed (yes, no screenshot)
```

---

### Example 2 — Delete Test Data After Each Test

```javascript
import { test, expect } from "@playwright/test";

let testUserId;

test.beforeEach(async ({ request }) => {
  // Create test user
  const response = await request.post("https://api.example.com/users", {
    data: { name: "Test", email: `user-${Date.now()}@example.com` }
  });
  testUserId = (await response.json()).id;
  console.log(`✅ User created: ${testUserId}`);
});

test("Test 1 - Update user", async ({ request }) => {
  const response = await request.put(`https://api.example.com/users/${testUserId}`, {
    data: { name: "Updated Name" }
  });
  expect(response.status()).toBe(200);
});

test("Test 2 - Delete user", async ({ request }) => {
  const response = await request.delete(`https://api.example.com/users/${testUserId}`);
  expect(response.status()).toBe(200);
});

test.afterEach(async ({ request }) => {
  // Har test ke baad user delete karo
  try {
    await request.delete(`https://api.example.com/users/${testUserId}`);
    console.log(`🗑️ User deleted: ${testUserId}`);
  } catch (e) {
    // Agar already deleted hai toh koi issue nahi
    console.log(`User already deleted`);
  }
});
```

Har test independent rahe — pehle test ka data doosre test ko affect na kare.

---

### Example 3 — Log Test Duration After Each Test

```javascript
import { test, expect } from "@playwright/test";

test("Test 1 - Fast test", async ({ page }) => {
  await page.goto("https://example.com");
  expect(await page.isVisible("text=Home")).toBeTruthy();
});

test("Test 2 - Slow test", async ({ page }) => {
  await page.goto("https://example.com");
  await page.waitForTimeout(3000);  // 3 second wait
  expect(await page.isVisible("text=Home")).toBeTruthy();
});

test.afterEach(async ({}, testInfo) => {
  // Test duration log karo
  const duration = testInfo.duration;
  console.log(`⏱️ ${testInfo.title} took ${duration}ms`);
  
  if (duration > 2000) {
    console.log(`⚠️ WARNING: Test took more than 2 seconds!`);
  }
});
```

**Output:**
```
⏱️ Test 1 - Fast test took 523ms
⏱️ Test 2 - Slow test took 3456ms
⚠️ WARNING: Test took more than 2 seconds!
```

## Summary Table — All 4 Hooks

| Hook | Runs | Frequency | Use Case |
|------|------|-----------|----------|
| `beforeAll` | Before all tests start | Once | Database connection, Server startup, Token fetch |
| `afterAll` | After all tests end | Once | Close database, Stop server, Delete temp files |
| `beforeEach` | Before every test | Per test | Login, Reset app state, Create test data |
| `afterEach` | After every test | Per test | Delete test data, Take screenshots, Log info |
