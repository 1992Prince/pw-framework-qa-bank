# Playwright Hooks — Part 2: Combining Hooks

---

## Q1. Can we use beforeAll/afterAll + beforeEach/afterEach together in same spec file?

**Answer:** Haan, bilkul! Tum ek hi spec file mein `beforeAll`, `afterAll`, `beforeEach`, aur `afterEach` — sabko saath use kar sakte ho. Playwright inhe ek specific order mein execute karta hai.

**Execution Order:**
```
beforeAll → beforeEach → test → afterEach → beforeEach → test → afterEach → ... → afterAll
```

### Example — All 4 Hooks Together

```javascript
import { test, expect } from "@playwright/test";

// ─── Global Variables ───
let dbConnection;
let testUserId;

// ─── Hook 1: beforeAll ───
test.beforeAll(async () => {
  // Ek baar: database connection open karo
  dbConnection = await connectToDatabase({ host: "localhost", port: 5432 });
  console.log("🟢 beforeAll — DB connected");
});

// ─── Hook 2: beforeEach ───
test.beforeEach(async ({ page }) => {
  // Har test se pehle: login karo
  await page.goto("https://example.com/login");
  await page.fill('input[name="email"]', "admin@example.com");
  await page.fill('input[name="password"]', "admin123");
  await page.click("button:has-text('Login')");
  await page.waitForURL("**/dashboard");
  console.log("🔵 beforeEach — User logged in");
});

// ─── Test 1 ───
test("TC001 - View dashboard", async ({ page }) => {
  console.log("▶ TC001 running");
  await expect(page).toHaveURL(/dashboard/);
});

// ─── Test 2 ───
test("TC002 - View profile", async ({ page }) => {
  console.log("▶ TC002 running");
  await page.click("text=Profile");
  await expect(page.locator("h1")).toHaveText("My Profile");
});

// ─── Hook 3: afterEach ───
test.afterEach(async ({ page }, testInfo) => {
  // Har test ke baad: screenshot lo agar fail hua
  if (testInfo.status !== "passed") {
    await page.screenshot({ path: `./screenshots/${testInfo.title}-failed.png` });
    console.log(`📸 afterEach — Screenshot saved for failed test`);
  }
  console.log(`🟡 afterEach — Test "${testInfo.title}" status: ${testInfo.status}`);
});

// ─── Hook 4: afterAll ───
test.afterAll(async () => {
  // Ek baar: database connection close karo
  await dbConnection.close();
  console.log("🔴 afterAll — DB disconnected");
});
```

### Console Output (2 tests ke liye):

```
🟢 beforeAll — DB connected

🔵 beforeEach — User logged in
▶ TC001 running
🟡 afterEach — Test "TC001 - View dashboard" status: passed

🔵 beforeEach — User logged in
▶ TC002 running
🟡 afterEach — Test "TC002 - View profile" status: passed

🔴 afterAll — DB disconnected
```

## Q2. Can we have Multiple beforeAll/afterAll and Multiple beforeEach/afterEach in same spec file? What is the execution order?

**Answer:** Haan! Ek hi spec file mein multiple `beforeAll`, multiple `afterAll`, multiple `beforeEach`, aur multiple `afterEach` rakh sakte ho. Playwright inhe **declaration order** (upar se neeche) mein execute karta hai.

### Rules:
| Hook | Multiple Allowed? | Execution Order |
|------|-------------------|-----------------|
| `beforeAll` | ✅ Yes | Top to bottom (declaration order) |
| `afterAll` | ✅ Yes | Top to bottom (declaration order) |
| `beforeEach` | ✅ Yes | Top to bottom (declaration order) |
| `afterEach` | ✅ Yes | Top to bottom (declaration order) |

---

### Example — Multiple of Each Hook

```javascript

  console.log("🔵 beforeEach #2 — Fill login credentials");
  await page.fill('input[name="email"]', "user@example.com");
  await page.fill('input[name="password"]', "pass123");
});

test.beforeEach(async ({ page }) => {
  console.log("🔵 beforeEach #3 — Click login button");
  await page.click("button:has-text('Login')");
  await page.waitForURL("**/dashboard");

// ─── Tests ───
test("TC001 - Dashboard visible", async ({ page }) => {
```

### Why Multiple Hooks Useful Hain?

**Separation of Concerns** — Har hook ek kaam karo:

```javascript
// ✅ GOOD — responsibilities alag hain
test.beforeEach(async ({ page }) => {
  await page.goto("https://example.com/login");  // navigation
});

test.beforeEach(async ({ page }) => {
  await login(page, "user@example.com", "pass");  // authentication
});

test.beforeEach(async ({ page }) => {
  await page.waitForSelector(".dashboard");  // wait for ready state
});
```

```javascript
// ❌ AVOID — sab kuch ek hi hook mein cramming
test.beforeEach(async ({ page }) => {
  await page.goto("https://example.com/login");
  await login(page, "user@example.com", "pass");
  await page.waitForSelector(".dashboard");
  await setupTestData();
  await clearOldLogs();
  // ... 50 more lines
});
```

## Q3. How Hooks Appear in Test Reports

**Answer:** Playwright ke HTML report mein hooks ka output test ke saath dikhta hai. Kuch important points:

### 1. Hooks Console Logs — `stdout` Mein Dikhte Hain

Jab `console.log()` hook ke andar use karo, woh test ke stdout section mein dikhta hai:

```javascript
test.beforeEach(async ({ page }) => {
  console.log("✅ Login complete");  // Report mein dikhega
});

test("TC001 - Dashboard", async ({ page }) => {
  console.log("▶ Test running");    // Report mein dikhega
  await expect(page).toHaveURL(/dashboard/);
});
```

**HTML Report mein TC001 ke andar:**
```
stdout
  ✅ Login complete        ← beforeEach ka log
  ▶ Test running          ← test ka log
```

---

### 2. Hook Failures — Test Ko Failed Mark Karte Hain

Agar `beforeEach` mein koi error aaye, toh woh test **failed** dikhega — even if test body mein koi problem nahi thi:

```javascript
test.beforeEach(async ({ page }) => {
  await page.goto("https://invalid-url-that-doesnt-exist.com");
  // ❌ Yeh fail hoga — aur TC001 bhi failed dikhega
});

test("TC001 - Dashboard", async ({ page }) => {
  // Yeh kabhi execute hi nahi hoga
  await expect(page).toHaveURL(/dashboard/);
});
```

**Report mein:**
```
❌ TC001 - Dashboard    FAILED
   Error in: Before Hooks
   Error: net::ERR_NAME_NOT_RESOLVED
```

---

### 3. Hook Errors Separately Labeled Hote Hain

Playwright report mein hooks ke errors clearly labeled hote hain:

```
❌ TC001 - Dashboard
   ├── Before Hooks        ← beforeEach error yahan dikhega
   │   └── Error: ...
   ├── Test Body           ← test body error yahan dikhega
   │   └── Error: ...
   └── After Hooks         ← afterEach error yahan dikhega
       └── Error: ...
```

---

### 4. testInfo Object Se Rich Reporting

`afterEach` mein `testInfo` use karke test report ko enrich kar sakte ho:

```javascript
import { test, expect } from "@playwright/test";

test.afterEach(async ({ page }, testInfo) => {
  // Test ka title
  console.log("Test name:", testInfo.title);

  // Pass/fail/skipped/timedOut
  console.log("Status:", testInfo.status);

  // Expected status (usually 'passed')
  console.log("Expected status:", testInfo.expectedStatus);

  // Test duration in ms
  console.log("Duration:", testInfo.duration, "ms");

  // Retry attempt number (0 = first run, 1 = first retry)
  console.log("Retry #:", testInfo.retry);

  // File path of the spec file
  console.log("Spec file:", testInfo.file);

  // ─── Attach screenshot to report on failure ───
  if (testInfo.status !== testInfo.expectedStatus) {
    const screenshot = await page.screenshot();
    await testInfo.attach("failure-screenshot", {
      body: screenshot,
      contentType: "image/png",
    });
    console.log("📸 Screenshot attached to report");
  }
});

test("TC001 - Login test", async ({ page }) => {
  await page.goto("https://example.com/login");
  await expect(page.locator("h1")).toHaveText("Login");
});
```

**Report mein TC001 ke andar attachment dikhega:**
```
TC001 - Login test    ✅ passed  (523ms)
  Attachments:
    📎 failure-screenshot  [only if failed]
  stdout:
    Test name: TC001 - Login test
    Status: passed
    Expected status: passed
    Duration: 523 ms
    Retry #: 0
    Spec file: tests/login.spec.js
```

### 5. Complete Real-World Example With Reporting

```javascript
import { test, expect } from "@playwright/test";

test.beforeAll(async () => {
  console.log("=== TEST SUITE STARTED ===");
});

test.beforeEach(async ({ page }, testInfo) => {
  console.log(`\n--- Starting: ${testInfo.title} ---`);
  await page.goto("https://example.com");
});

test("TC001 - Homepage loads", async ({ page }) => {
  await expect(page).toHaveTitle(/Example Domain/);
});

test("TC002 - Heading visible", async ({ page }) => {
  await expect(page.locator("h1")).toBeVisible();
});

test.afterEach(async ({ page }, testInfo) => {
  // Always attach screenshot (pass ya fail)
  const screenshot = await page.screenshot({ fullPage: true });
  await testInfo.attach(`${testInfo.title}-screenshot`, {
    body: screenshot,
    contentType: "image/png",
  });

  // Status log
  const icon = testInfo.status === "passed" ? "✅" : "❌";
  console.log(`${icon} ${testInfo.title} — ${testInfo.status} (${testInfo.duration}ms)`);
});

test.afterAll(async () => {
  console.log("=== TEST SUITE ENDED ===");
});
```

**HTML Report Output:**

```
=== TEST SUITE STARTED ===

--- Starting: TC001 - Homepage loads ---
✅ TC001 - Homepage loads — passed (412ms)
  Attachments: TC001 - Homepage loads-screenshot.png

--- Starting: TC002 - Heading visible ---
✅ TC002 - Heading visible — passed (389ms)
  Attachments: TC002 - Heading visible-screenshot.png

=== TEST SUITE ENDED ===
```

## Summary

| Concept | Key Point |
|---------|-----------|
| Combining hooks | `beforeAll` → `beforeEach` → test → `afterEach` → `afterAll` |
| Multiple hooks allowed | Haan — declaration order mein execute hote hain |
| Hook failure | Woh test ko fail karta hai, report mein "Before/After Hooks" label ke saath |
| `testInfo` in afterEach | Title, status, duration, retry info milti hai |
| `testInfo.attach()` | Screenshots/files ko report mein attach karo |
