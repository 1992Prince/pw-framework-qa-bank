# Data Driven Testing with CSV in Playwright

## Why CSV?

In real projects, many times test data comes from business teams, QA leads, or clients — and they usually share it in Excel or CSV format because that's what non-technical people are comfortable with. CSV (Comma Separated Values) is a simple text file where each row is a record and columns are separated by commas. Unlike JSON, CSV is not natively supported by JavaScript — we can't just import it like a JSON file. We need a parser library to read and convert CSV data into JavaScript objects.

---

## Why PapaParse Library?

JavaScript/TypeScript cannot read CSV files directly. If we read a CSV file as plain text, we get a raw string like:

```
id,name,email,password
TC001,Sarika,sarika@example.com,password123
TC002,John,john@example.com,password456
```

This is just a string — not an array of objects we can loop through. We need something to parse (break down) this string into structured data. That's what PapaParse does — it takes raw CSV text and converts it into a JavaScript array of objects, just like what we get when we import a JSON file.

PapaParse is one of the most popular CSV parsing libraries in the JavaScript ecosystem — fast, lightweight, and handles edge cases like commas inside quotes, empty lines, headers, etc.

## Complete Code — Line by Line Explanation

```typescript
import { test, expect } from "@playwright/test";
```

Named import — imports `test` and `expect` from the Playwright library. `test` is used to define test cases, `expect` is used for assertions. Curly braces because these are named exports.

---

```typescript
import Papa from "papaparse";
```

Default import — imports the PapaParse library. The library exports one main object as default, so no curly braces. We named it `Papa` (convention), but we could call it anything — `csvParser`, `PapaParse`, etc.

Why do we need this? JavaScript cannot understand CSV format on its own. PapaParse converts raw CSV text into a JavaScript array of objects that we can loop through.

---

```typescript
import fs from "fs";
```

Default import — imports Node.js's built-in `fs` (File System) module. This module lets us read and write files from disk. We need it because we have to read the CSV file's content from the file system before PapaParse can parse it.

**Key point:** `fs` is a built-in Node.js module — no need to install it. It's always available in any Node.js project.

Why do we need this? PapaParse doesn't read files from disk — it only parses CSV strings. So first we read the file using `fs`, get the raw text string, and then give that string to PapaParse for parsing.

---

```typescript
function readCSV(filePath) {
```

A regular JavaScript function that takes a file path as input (like `"testdata/registration.csv"`) and returns parsed CSV data as an array of objects. We created this as a reusable function so we can call it for any CSV file — registration data, login data, payment data, etc.

---

```typescript
  const csvData = fs.readFileSync(filePath, "utf-8");
```

What it does: Reads the CSV file from disk and stores its entire content as a plain text string in the variable `csvData`.

- `fs.readFileSync` — reads a file synchronously (blocks execution until file is fully read). We use the sync version because we need the data before tests start — there's nothing else to do in parallel.
- `filePath` — the path we passed as argument, e.g., `"testdata/registration.csv"`.
- `"utf-8"` — the encoding format. Tells Node.js to read the file as normal text (not binary).

What `csvData` looks like after this line:

```
"id,name,email,password\nTC001,Sarika,sarika@example.com,password123\nTC002,John,john@example.com,password456\nTC003,Alice,alice@example.com,password789"
```

Just a raw string — not usable for testing yet. We need PapaParse to convert this into structured objects.

---

```typescript
  const parsedData = Papa.parse(csvData, {
    header: true,
    skipEmptyLines: true,
  });
```

What it does: PapaParse takes the raw CSV string and converts it into structured JavaScript data.

- `Papa.parse()` — the main parsing function of PapaParse. First argument is the CSV string, second is an options object.
- `header: true` — tells PapaParse to treat the first row as column headers (not data). So `id`, `name`, `email`, `password` become the keys of each object. Without this, PapaParse would return arrays of values instead of objects — much harder to work with.
- `skipEmptyLines: true` — if the CSV file has any blank lines (common when editing in Excel), skip them instead of creating empty records.

What `parsedData` looks like after this line:

```javascript
{
  data: [
    { id: "TC001", name: "Sarika", email: "sarika@example.com", password: "password123" },
    { id: "TC002", name: "John",   email: "john@example.com",   password: "password456" },
    { id: "TC003", name: "Alice",  email: "alice@example.com",  password: "password789" }
  ],
  errors: [],
  meta: { delimiter: ",", linebreak: "\n", aborted: false, truncated: false, fields: ["id","name","email","password"] }
}
```

PapaParse returns a result object with three properties:

- `data` — the actual parsed data as an array of objects (this is what we need)
- `errors` — any parsing errors (empty if everything is fine)
- `meta` — metadata about the CSV (delimiter used, fields found, etc.)

---

```typescript
  return parsedData.data;
```

We return only `parsedData.data` — the array of objects. We don't need `errors` or `meta` for our tests. After this return, the function gives us exactly the same structure as a JSON import:

```javascript
[
  { id: "TC001", name: "Sarika", email: "sarika@example.com", password: "password123" },
  { id: "TC002", name: "John",   email: "john@example.com",   password: "password456" },
  { id: "TC003", name: "Alice",  email: "alice@example.com",  password: "password789" }
]
```


Calls our function with the CSV file path. The function reads the file, parses it, and returns the array of objects. Now `testdata` holds exactly the same data structure as the JSON approach — an array of objects. From this point onwards, the code is identical to the JSON version.

`for...of` loop — iterates through the `testdata` array. Each iteration, `data` holds one record object. 3 records = 3 iterations = 3 test cases.

- Iteration 1: `data = { id: "TC001", name: "Sarika", ... }`

- Iteration 3: `data = { id: "TC003", name: "Alice", ... }`

Creates a Playwright test for each data record. The test name includes `data.id` to make it unique — `Test TC001`, `Test TC002`, `Test TC003`. If names aren't unique, Playwright throws an error.

## The Complete Flow — What Happens Step by Step

```
1. fs.readFileSync reads registration.csv → raw text string
                    ↓
2. Papa.parse() converts raw string → result object with data, errors, meta
                    ↓
3. return parsedData.data → array of 3 objects (same as JSON import)
                    ↓
4. testdata = array of 3 objects
                    ↓
5. for...of loop creates 3 separate tests → Test TC001, Test TC002, Test TC003
                    ↓
6. Each test runs independently with its own data record
                    ↓
7. Report shows 3 separate tests — TC001 fails, TC002 passes, TC003 passes
```

## JSON vs CSV Approach — Comparison

| Aspect                    | JSON                                          | CSV                                                                     |
| ------------------------- | --------------------------------------------- | ----------------------------------------------------------------------- |
| Import                    | Direct — `import data from "file.json"`       | Needs `fs` + PapaParse                                                  |
| Extra library needed?     | No                                            | Yes — `papaparse`                                                       |
| File readability          | Good for developers                           | Good for non-technical people (opens in Excel)                          |
| When to use               | Data is created/managed by developers         | Data comes from business teams, QA leads, clients in spreadsheet format |
| Result after parsing      | Array of objects                              | Array of objects (same as JSON)                                         |
| Code after data loading   | Identical                                     | Identical                                                               |

> **Key takeaway:** Whether we use JSON or CSV, after loading the data we get the same structure — an array of objects. The only difference is how we read and parse the file. The test logic remains exactly the same.

---

## Summary

- CSV files can't be imported directly in JavaScript like JSON — we need a parser.
- `fs` module (built-in Node.js) reads the file from disk as a raw text string.
- PapaParse converts that raw CSV string into a JavaScript array of objects.
- `header: true` tells PapaParse to use the first row as keys (column names become object property names).
- `skipEmptyLines: true` prevents empty rows from becoming empty records.
- `Papa.parse()` returns a result object — we only use `.data` from it (the array of records).
- After parsing, the data structure is identical to JSON — same array of objects, same `for...of` loop, same test code.
- Install PapaParse with `npm install -D papaparse`.
- Use CSV when data comes from non-technical stakeholders in spreadsheet format. Use JSON when developers manage the data.

## Complete Spec File

```javascript
import { test, expect } from "@playwright/test";
import Papa from "papaparse"; // import papaparse library to parse CSV files
import fs from "fs"; // import fs module to read files


function readCSV(filePath){
  const csvData = fs.readFileSync(filePath, "utf-8"); // read the CSV file
  const parsedData = Papa.parse(csvData, {
    header: true, // treat the first row as header
    skipEmptyLines: true, // skip empty lines in the CSV file
  });
  return parsedData.data; // return the parsed data as an array of objects
}

const testdata = readCSV("testdata/registration.csv"); // read the test data from the CSV file

for (const data of testdata) {
  test(`Test ${data.id}`, async ({ page }) => {
    await page.goto("https://gauravkhurana.com/practise-api/ui/index.html#/practice");

    await page.getByTestId("form-email").fill(data.email);
    await page.getByTestId("form-password").fill(data.password);
    await page.getByTestId("form-confirm-password").fill(data.name);

    if(data.id === 'TC001'){
      // this assertion will fail for TC001 as the expected value is "123" but the actual value is "TC001"
      await expect(data.id).toBe("123"); 
    }
    
    await page.waitForTimeout(3000); // hard wait to see the filled data in the form before the next test runs
  });
}
```
