# Playwright Hooks — Part 3: Test Grouping

---

## What is test.describe()? Why Group Tests?

### What Is It?

`test.describe()` ek container hai jo related tests ko ek group mein organize karta hai. Jaise ek folder mein related files rakhte ho, waise hi `describe` mein related tests rakhte ho.

### Why Group Tests?

| Problem (Without Groups) | Solution (With Groups) |
|--------------------------|------------------------|
| Sare tests ek flat list mein — confusing | Tests logically organized — easy to read |
| "Login test", "Cart test" sab mixed | Login tests alag, Cart tests alag |
| Ek group ke liye alag hooks nahi laga sakte | Har group ka apna `beforeEach`/`afterAll` ho sakta hai |
| Report mein structure nahi dikhta | Report mein grouped hierarchy dikhti hai |

### Real-World Analogy

Soch ek school ka timetable:
- **Subject: Math** — Algebra, Geometry, Calculus (group 1)
- **Subject: Science** — Physics, Chemistry, Biology (group 2)

Sabko ek list mein nahi daalte — subjects ke groups bante hain. `test.describe()` wahi karta hai.

---

## Syntax aur Basic Example

### Syntax

```javascript
test.describe("Group Name", () => {
  // hooks (optional)
  test.beforeEach(async ({ page }) => { /* ... */ });

  // tests
  test("test name", async ({ page }) => { /* ... */ });
});
```

### Basic Example — Login Tests Group

```javascript
import { test, expect } from "@playwright/test";

test.describe("Login Tests", () => {
  test.beforeEach(async ({ page }) => {
    console.log("🔵 beforeEach — Navigate to login page");
    await page.goto("https://example.com/login");
  });

  test("TC001 - Valid login", async ({ page }) => {
    console.log("▶ TC001 — Valid login");
    await page.fill('input[name="email"]', "user@example.com");
    await page.fill('input[name="password"]', "pass123");
    await page.click("button:has-text('Login')");
    await expect(page).toHaveURL(/dashboard/);
  });

  test("TC002 - Invalid password", async ({ page }) => {
    console.log("▶ TC002 — Invalid password");
    await page.fill('input[name="email"]', "user@example.com");
    await page.fill('input[name="password"]', "wrongpass");
    await page.click("button:has-text('Login')");
    await expect(page.locator(".error-msg")).toHaveText("Invalid credentials");
  });

  test("TC003 - Empty fields", async ({ page }) => {
    console.log("▶ TC003 — Empty fields");
    await page.click("button:has-text('Login')");
    await expect(page.locator(".error-msg")).toHaveText("Fields are required");
  });
});
```

**Console Output:**
```
🔵 beforeEach — Navigate to login page
▶ TC001 — Valid login

🔵 beforeEach — Navigate to login page
▶ TC002 — Invalid password

🔵 beforeEach — Navigate to login page
▶ TC003 — Empty fields
```

**HTML Report mein:**
```
Login Tests
  ✅ TC001 - Valid login
  ✅ TC002 - Invalid password
  ✅ TC003 - Empty fields
```

## Multiple Groups in One Spec File

Ek spec file mein multiple `describe` groups rakh sakte ho — har group ka apna `beforeEach`, `afterEach`, etc. hoga.

### Key Rule — Hooks Are Scoped to Their Group

- Group A ka `beforeEach` sirf Group A ke tests ke liye chalega
- Group B ka `beforeEach` sirf Group B ke tests ke liye chalega
- Dono groups ke hooks ek doosre ko affect nahi karte

### Example — Login + Registration Groups

```javascript
import { test, expect } from "@playwright/test";

// ─── Group 1: Login Tests ───
test.describe("Login Tests", () => {
  test.beforeAll(async () => {
    console.log("🟢 [Login] beforeAll — Setup login test data");
  });

  test.beforeEach(async ({ page }) => {
    console.log("🔵 [Login] beforeEach — Go to login page");
    await page.goto("https://example.com/login");
  });

  test("TC001 - Valid login", async ({ page }) => {
    console.log("▶ TC001 running");
    await page.fill('input[name="email"]', "user@example.com");
    await page.fill('input[name="password"]', "pass123");
    await page.click("button:has-text('Login')");
    await expect(page).toHaveURL(/dashboard/);
  });

  test("TC002 - Invalid login", async ({ page }) => {
    console.log("▶ TC002 running");
    await page.fill('input[name="email"]', "wrong@example.com");
    await page.fill('input[name="password"]', "wrongpass");
    await page.click("button:has-text('Login')");
    await expect(page.locator(".error")).toBeVisible();
  });

  test.afterAll(async () => {
    console.log("🔴 [Login] afterAll — Cleanup login test data");
  });
});

// ─── Group 2: Registration Tests ───
test.describe("Registration Tests", () => {
  test.beforeAll(async () => {
    console.log("🟢 [Registration] beforeAll — Setup registration test data");
  });

  test.beforeEach(async ({ page }) => {
    console.log("🔵 [Registration] beforeEach — Go to registration page");
    await page.goto("https://example.com/register");
  });

  test("TC001 - Valid registration", async ({ page }) => {
    console.log("▶ TC001 running");
    await page.fill('input[name="name"]', "John Doe");
    await page.fill('input[name="email"]', `user-${Date.now()}@example.com`);
    await page.fill('input[name="password"]', "newpass123");
    await page.click("button:has-text('Register')");
    await expect(page).toHaveURL(/success/);
  });

  test("TC002 - Duplicate email", async ({ page }) => {
    console.log("▶ TC002 running");
    await page.fill('input[name="email"]', "existing@example.com");
    await page.fill('input[name="password"]', "pass123");
    await page.click("button:has-text('Register')");
    await expect(page.locator(".error")).toHaveText("Email already exists");
  });

  test.afterAll(async () => {
    console.log("🔴 [Registration] afterAll — Cleanup registration test data");
  });
});
```

**Console Output:**
```
🟢 [Login] beforeAll — Setup login test data

🔵 [Login] beforeEach — Go to login page
▶ TC001 running

🔵 [Login] beforeEach — Go to login page
▶ TC002 running

🔴 [Login] afterAll — Cleanup login test data

🟢 [Registration] beforeAll — Setup registration test data

🔵 [Registration] beforeEach — Go to registration page
▶ TC001 running

🔵 [Registration] beforeEach — Go to registration page
▶ TC002 running

🔴 [Registration] afterAll — Cleanup registration test data
```

**HTML Report mein:**
```
Login Tests
  ✅ TC001 - Valid login
  ✅ TC002 - Invalid login

Registration Tests
  ✅ TC001 - Valid registration
  ✅ TC002 - Duplicate email
```

## Independent Tests + Grouped Tests in Same Spec File

Ek spec file mein **top-level (independent) tests** aur **grouped tests** — dono saath rakh sakte ho.

### Execution Order Rule

```
Top-level beforeAll  →  Top-level beforeEach  →  [Group beforeAll]
→ [Group beforeEach]  →  Group Test  →  [Group afterEach]
→  Top-level afterEach  →  [Group afterAll]  →  Top-level afterAll
```

Top-level hooks **wrap around** group hooks — group hooks andar hote hain.

### Example

```javascript
import { test, expect } from "@playwright/test";

// ─── Top-Level Hooks (apply to ALL tests including groups) ───
test.beforeAll(async () => {
  console.log("🟢 [TOP] beforeAll — Global setup");
});

test.beforeEach(async ({ page }) => {
  console.log("🔵 [TOP] beforeEach — Runs before EVERY test");
});

test.afterEach(async ({}, testInfo) => {
  console.log(`🟡 [TOP] afterEach — "${testInfo.title}" done`);
});

test.afterAll(async () => {
  console.log("🔴 [TOP] afterAll — Global teardown");
});

// ─── Independent Top-Level Test ───
test("TC000 - Homepage check", async ({ page }) => {
  console.log("▶ TC000 — Independent test (no group)");
  await page.goto("https://example.com");
  await expect(page).toHaveTitle(/Example/);
});

// ─── Group 1: Login Tests ───
test.describe("Login Tests", () => {
  test.beforeEach(async ({ page }) => {
    console.log("🔵 [Login] beforeEach — Go to login page");
    await page.goto("https://example.com/login");
  });

  test.afterEach(async () => {
    console.log("🟡 [Login] afterEach — Login group cleanup");
  });

  test("TC001 - Valid login", async ({ page }) => {
    console.log("▶ TC001 running");
    await page.fill('input[name="email"]', "user@example.com");
    await page.fill('input[name="password"]', "pass123");
    await page.click("button:has-text('Login')");
    await expect(page).toHaveURL(/dashboard/);
  });

  test("TC002 - Invalid login", async ({ page }) => {
    console.log("▶ TC002 running");
    await expect(page.locator(".error")).not.toBeVisible();
  });
});

// ─── Group 2: Cart Tests ───
test.describe("Cart Tests", () => {
  test.beforeEach(async ({ page }) => {
    console.log("🔵 [Cart] beforeEach — Go to cart page");
    await page.goto("https://example.com/cart");
  });

  test("TC003 - Cart is empty", async ({ page }) => {
    console.log("▶ TC003 running");
    await expect(page.locator(".cart-items")).toBeEmpty();
  });
});
```

**Console Output (full execution):**
```
🟢 [TOP] beforeAll — Global setup

── TC000 ──
🔵 [TOP] beforeEach — Runs before EVERY test
▶ TC000 — Independent test (no group)
🟡 [TOP] afterEach — "TC000 - Homepage check" done

── TC001 (Login Group) ──
🔵 [TOP] beforeEach — Runs before EVERY test     ← top-level first
🔵 [Login] beforeEach — Go to login page          ← group second
▶ TC001 running
🟡 [Login] afterEach — Login group cleanup        ← group first
🟡 [TOP] afterEach — "TC001 - Valid login" done   ← top-level second

── TC002 (Login Group) ──
🔵 [TOP] beforeEach — Runs before EVERY test
🔵 [Login] beforeEach — Go to login page
▶ TC002 running
🟡 [Login] afterEach — Login group cleanup
🟡 [TOP] afterEach — "TC002 - Invalid login" done

── TC003 (Cart Group) ──
🔵 [TOP] beforeEach — Runs before EVERY test
🔵 [Cart] beforeEach — Go to cart page
▶ TC003 running
🟡 [TOP] afterEach — "TC003 - Cart is empty" done

🔴 [TOP] afterAll — Global teardown
```

## Test Name Uniqueness Within a Spec File (Across Groups)

### The Rule

Playwright test ko **full path** se identify karta hai:
```
Spec File > Group Name > Test Name
```

Isliye **same test name alag groups mein reuse kar sakte ho** — koi conflict nahi hoga.

### Example — Same Names, Different Groups

```javascript
import { test, expect } from "@playwright/test";

test.describe("Login Tests", () => {
  // ✅ "TC001 - Valid input" — Login group mein
  test("TC001 - Valid input", async ({ page }) => {
    console.log("▶ [Login] TC001 running");
    await page.goto("https://example.com/login");
    await page.fill('input[name="email"]', "user@example.com");
    await page.fill('input[name="password"]', "pass123");
    await page.click("button:has-text('Login')");
    await expect(page).toHaveURL(/dashboard/);
  });

  // ✅ "TC002 - Invalid input" — Login group mein
  test("TC002 - Invalid input", async ({ page }) => {
    console.log("▶ [Login] TC002 running");
    await page.goto("https://example.com/login");
    await page.fill('input[name="email"]', "wrong@example.com");
    await page.fill('input[name="password"]', "wrong");
    await page.click("button:has-text('Login')");
    await expect(page.locator(".error")).toBeVisible();
  });
});

test.describe("Registration Tests", () => {
  // ✅ Same name "TC001 - Valid input" — Registration group mein — NO CONFLICT
  test("TC001 - Valid input", async ({ page }) => {
    console.log("▶ [Registration] TC001 running");
    await page.goto("https://example.com/register");
    await page.fill('input[name="name"]', "Jane Doe");
    await page.fill('input[name="email"]', `new-${Date.now()}@example.com`);
    await page.fill('input[name="password"]', "newpass123");
    await page.click("button:has-text('Register')");
    await expect(page).toHaveURL(/success/);
  });

  // ✅ Same name "TC002 - Invalid input" — Registration group mein — NO CONFLICT
  test("TC002 - Invalid input", async ({ page }) => {
    console.log("▶ [Registration] TC002 running");
    await page.goto("https://example.com/register");
    await page.fill('input[name="email"]', "existing@example.com");
    await page.click("button:has-text('Register')");
    await expect(page.locator(".error")).toBeVisible();
  });
});
```

**HTML Report mein — clearly differentiated:**
```
Login Tests
  ✅ TC001 - Valid input        ← "Login Tests > TC001 - Valid input"
  ✅ TC002 - Invalid input      ← "Login Tests > TC002 - Invalid input"

Registration Tests
  ✅ TC001 - Valid input        ← "Registration Tests > TC001 - Valid input"
  ✅ TC002 - Invalid input      ← "Registration Tests > TC002 - Invalid input"
```

Playwright internally full path use karta hai — toh dono `TC001` alag entities hain.

### ❌ What To Avoid — Same Name at Same Level

```javascript
// ❌ AVOID — same name, same level (no group) = confusion in reports
test("TC001 - Login", async ({ page }) => { /* ... */ });
test("TC001 - Login", async ({ page }) => { /* ... */ }); // duplicate!
```

Playwright yeh run kar lega lekin report mein dono same naam se dikhenge — debugging mushkil ho jaayegi. Always use groups ya unique names.

---

## Summary

| Topic | Key Point |
|-------|-----------|
| `test.describe()` | Related tests ko logically group karta hai |
| Hooks scope | Group ke andar ke hooks sirf us group ke tests pe apply hote hain |
| Multiple groups | Ek file mein multiple groups allowed — har group isolated |
| Independent + grouped | Top-level hooks sab tests ko wrap karte hain (including grouped) |
| Test name uniqueness | Same name alag groups mein reuse kar sakte ho — full path se identify hota hai |
