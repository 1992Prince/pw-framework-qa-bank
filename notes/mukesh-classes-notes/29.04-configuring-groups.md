# Playwright Hooks — Part 4: Group Configuration

---

## Overview — Group Ko Configure Karne Ke 3 Tarike

```
1. test.use()               → fixtures/settings override karo (viewport, colorScheme, locale, etc.)
2. test.describe.configure() → parallelism aur retries control karo (serial/parallel mode)
3. Tags on describe/test    → tests ko label karo (smoke, regression, etc.) — filter karke run karo
```

---

## 1. test.use() Inside test.describe()

### Kya Hota Hai?

`test.use()` ek configuration override hai. Jab ise `test.describe()` ke **andar** likhte ho, toh woh sirf us group ke tests pe apply hota hai — baaki tests affect nahi hote.

Jab `test.use()` ko **top-level** (describe ke bahar) likhte ho, toh poori spec file ke sabhi tests pe apply hota hai.

### Syntax

```javascript
test.describe("Group Name", () => {
  test.use({
    viewport: { width: 375, height: 667 },   // Mobile viewport
    colorScheme: "dark",                      // Dark mode
    locale: "hi-IN",                          // Hindi locale
    // ... aur bhi options
  });

  test("test 1", async ({ page }) => { /* ... */ });
  test("test 2", async ({ page }) => { /* ... */ });
});
```

`test.use()` ko `describe` block ke andar **sabse upar** likhna chahiye — hooks aur tests se pehle.

---

### Common Options for test.use()

| Option | Values | Purpose |
|--------|--------|---------|
| `viewport` | `{ width, height }` | Browser window size set karo |
| `colorScheme` | `"light"`, `"dark"`, `"no-preference"` | Color theme |
| `locale` | `"en-US"`, `"hi-IN"`, `"fr-FR"` | Browser language |
| `timezoneId` | `"Asia/Kolkata"`, `"UTC"` | Timezone |
| `geolocation` | `{ latitude, longitude }` | GPS location |
| `permissions` | `["geolocation"]`, `["camera"]` | Browser permissions |
| `userAgent` | string | Custom user agent |
| `ignoreHTTPSErrors` | `true`/`false` | SSL errors ignore karo |
| `baseURL` | string | Base URL for `page.goto()` |

---

### Example 1 — Viewport Configuration

```javascript
import { test, expect } from "@playwright/test";

// ─── Group 1: Desktop Tests ───
test.describe("Desktop Tests", () => {
  test.use({ viewport: { width: 1920, height: 1080 } });

  test("TC001 - Desktop nav visible", async ({ page }) => {
    console.log("▶ TC001 — Desktop viewport: 1920x1080");
    await page.goto("https://example.com");
    // Desktop pe nav bar fully visible hoga
    await expect(page.locator("nav")).toBeVisible();
  });

  test("TC002 - Desktop sidebar visible", async ({ page }) => {
    console.log("▶ TC002 — Desktop viewport: 1920x1080");
    await page.goto("https://example.com/dashboard");
    await expect(page.locator(".sidebar")).toBeVisible();
  });
});

// ─── Group 2: Mobile Tests ───
test.describe("Mobile Tests", () => {
  test.use({ viewport: { width: 375, height: 667 } });

  test("TC001 - Mobile hamburger menu visible", async ({ page }) => {
    console.log("▶ TC001 — Mobile viewport: 375x667");
    await page.goto("https://example.com");
    // Mobile pe hamburger icon visible hoga, nav bar hidden
    await expect(page.locator(".hamburger-menu")).toBeVisible();
    await expect(page.locator("nav")).not.toBeVisible();
  });

  test("TC002 - Mobile footer visible", async ({ page }) => {
    console.log("▶ TC002 — Mobile viewport: 375x667");
    await page.goto("https://example.com");
    await expect(page.locator("footer")).toBeVisible();
  });
});
```

Yahan:
- `Desktop Tests` group ke dono tests — 1920x1080 viewport pe chalenge
- `Mobile Tests` group ke dono tests — 375x667 viewport pe chalenge

### Example 2 — Dark Mode vs Light Mode

```javascript
import { test, expect } from "@playwright/test";

test.describe("Light Mode Tests", () => {
  test.use({ colorScheme: "light" });

  test("TC001 - Light mode background color", async ({ page }) => {
    console.log("▶ TC001 — colorScheme: light");
    await page.goto("https://example.com");
    const bgColor = await page.evaluate(() =>
      window.getComputedStyle(document.body).backgroundColor
    );
    // Light mode mein white background hona chahiye
    expect(bgColor).toBe("rgb(255, 255, 255)");
  });
});

test.describe("Dark Mode Tests", () => {
  test.use({ colorScheme: "dark" });

  test("TC001 - Dark mode background color", async ({ page }) => {
    console.log("▶ TC001 — colorScheme: dark");
    await page.goto("https://example.com");
    const bgColor = await page.evaluate(() =>
      window.getComputedStyle(document.body).backgroundColor
    );
    // Dark mode mein dark background hona chahiye
    expect(bgColor).toBe("rgb(18, 18, 18)");
  });
});
```

### Example 3 — Locale + Timezone Configuration

```javascript
import { test, expect } from "@playwright/test";

test.describe("Indian Locale Tests", () => {
  test.use({
    locale: "hi-IN",
    timezoneId: "Asia/Kolkata",
  });

  test("TC001 - Date format in Indian locale", async ({ page }) => {
    console.log("▶ TC001 — locale: hi-IN, timezone: Asia/Kolkata");
    await page.goto("https://example.com/profile");
    // Date dd/mm/yyyy format mein dikhni chahiye
    const dateText = await page.locator(".member-since").textContent();
    expect(dateText).toMatch(/\d{2}\/\d{2}\/\d{4}/);
  });

test.describe("US Locale Tests", () => {
  test.use({
    locale: "en-US",
    timezoneId: "America/New_York",
  });

  test("TC001 - Date format in US locale", async ({ page }) => {
    console.log("▶ TC001 — locale: en-US, timezone: America/New_York");
    await page.goto("https://example.com/profile");
    // Date mm/dd/yyyy format mein dikhni chahiye
    const dateText = await page.locator(".member-since").textContent();
    expect(dateText).toMatch(/\d{2}\/\d{2}\/\d{4}/);
  });
});
```

## 2. test.describe.configure() — Parallelism aur Retries Control

### Kya Hota Hai?

By default Playwright tests parallel chalata hai. Lekin kabhi kabhi chahte ho ki ek group ke tests **ek ke baad ek** (serial) chalein — jaise ek workflow ke steps jo order pe depend karte hain.

`test.describe.configure()` se tum group-level pe yeh control kar sakte ho.

### Syntax

```javascript
test.describe("Group Name", () => {
  test.describe.configure({
    mode: "serial",   // "serial" | "parallel" (default)
    retries: 2,       // fail hone pe kitni baar retry karo
    timeout: 60000,   // is group ke tests ka timeout (ms)
  });

  test("step 1", async ({ page }) => { /* ... */ });
  test("step 2", async ({ page }) => { /* ... */ });
});
```

### Options

| Option | Values | Purpose |
|--------|--------|---------|
| `mode` | `"serial"`, `"parallel"` | Tests ek ke baad ek ya saath chalein |
| `retries` | number | Fail tests ko retry karo |
| `timeout` | number (ms) | Group ke tests ka default timeout |

---

### Example — Serial Mode (Order-Dependent Tests)

```javascript
import { test, expect } from "@playwright/test";

// Yeh tests ek checkout workflow ke steps hain — order important hai
test.describe("Checkout Workflow", () => {
  test.describe.configure({ mode: "serial" });

  test("Step 1 - Add item to cart", async ({ page }) => {
    console.log("▶ Step 1 — Add to cart");
    await page.goto("https://example.com/products/1");
    await page.click("button:has-text('Add to Cart')");
    await expect(page.locator(".cart-count")).toHaveText("1");
  });

  test("Step 2 - Go to cart", async ({ page }) => {
    console.log("▶ Step 2 — View cart");
    await page.goto("https://example.com/cart");
    await expect(page.locator(".cart-item")).toHaveCount(1);
  });

  test("Step 3 - Checkout", async ({ page }) => {
    console.log("▶ Step 3 — Checkout");
    await page.goto("https://example.com/checkout");
    await page.fill('input[name="card"]', "4111111111111111");
    await page.click("button:has-text('Place Order')");
    await expect(page).toHaveURL(/order-confirmation/);
  });
});
```

Serial mode mein agar Step 1 fail ho gaya, toh Step 2 aur Step 3 automatically skip ho jaate hain — waste of time nahi hota.

---

### Example — Retries Configuration

```javascript
import { test, expect } from "@playwright/test";

test.describe("Flaky API Tests", () => {
  test.describe.configure({ retries: 2 });  // Har test 2 baar retry hoga

  test("TC001 - Fetch user data", async ({ request }) => {
    console.log("▶ TC001 — API call (might be flaky)");
    const response = await request.get("https://api.example.com/users/1");
    expect(response.status()).toBe(200);
  });

  test("TC002 - Create order", async ({ request }) => {
    console.log("▶ TC002 — Create order API");
    const response = await request.post("https://api.example.com/orders", {
      data: { productId: 1, quantity: 2 },
    });
    expect(response.status()).toBe(201);
  });
});
```

## 3. Tags for Groups aur Individual Tests

### Kya Hain Tags?

Tags ek **label** hain jo tum group ya test pe lagate ho. Baad mein `--grep` flag se sirf woh tests run kar sakte ho jinpe woh tag hai.

Use Case:
- `@smoke` — sirf fast, critical tests run karo (CI/CD ke liye)
- `@regression` — full test suite run karo (nightly build ke liye)
- `@login` — sirf login-related tests run karo
- `@slow` — heavy tests alag se run karo

---

### Syntax — Group Level Tag

```javascript
// Poore group ko ek tag do
test.describe("Login Tests", { tag: "@smoke" }, () => {
  test("TC001 - Valid login", async ({ page }) => { /* ... */ });
  test("TC002 - Invalid login", async ({ page }) => { /* ... */ });
});
```

### Syntax — Individual Test Tag

```javascript
// Ek test ko tag do
test("TC001 - Valid login", { tag: "@smoke" }, async ({ page }) => {
  // ...
});

// Multiple tags ek test pe
test("TC002 - Full flow", { tag: ["@smoke", "@regression"] }, async ({ page }) => {
  // ...
});
```

---

### Example — Group Level Tags

```javascript
import { test, expect } from "@playwright/test";

// ─── Smoke Tests Group ───
test.describe("Login Smoke Tests", { tag: "@smoke" }, () => {
  test("TC001 - Valid login loads dashboard", async ({ page }) => {
    console.log("▶ [smoke] TC001 running");
    await page.goto("https://example.com/login");
    await page.fill('input[name="email"]', "user@example.com");
    await page.fill('input[name="password"]', "pass123");
    await page.click("button:has-text('Login')");
    await expect(page).toHaveURL(/dashboard/);
  });

  test("TC002 - Homepage loads", async ({ page }) => {
    console.log("▶ [smoke] TC002 running");
    await page.goto("https://example.com");
    await expect(page).toHaveTitle(/Example/);
  });
});

// ─── Regression Tests Group ───
test.describe("Login Regression Tests", { tag: "@regression" }, () => {
  test("TC001 - Invalid email format", async ({ page }) => {
    console.log("▶ [regression] TC001 running");
    await page.goto("https://example.com/login");
    await page.fill('input[name="email"]', "notanemail");
    await page.click("button:has-text('Login')");
    await expect(page.locator(".error")).toHaveText("Invalid email format");
  });

  test("TC002 - Password too short", async ({ page }) => {
    console.log("▶ [regression] TC002 running");
    await page.goto("https://example.com/login");
    await page.fill('input[name="email"]', "user@example.com");
    await page.fill('input[name="password"]', "ab");
    await page.click("button:has-text('Login')");
    await expect(page.locator(".error")).toHaveText("Password too short");
  });

  test("TC003 - Account locked after 5 attempts", async ({ page }) => {
    console.log("▶ [regression] TC003 running");
    // 5 failed attempts simulate karo
    for (let i = 0; i < 5; i++) {
      await page.goto("https://example.com/login");
      await page.fill('input[name="email"]', "user@example.com");
      await page.fill('input[name="password"]', "wrongpass");
      await page.click("button:has-text('Login')");
    }
    await expect(page.locator(".error")).toHaveText("Account locked");
  });
});
```

**Run karne ka tarika:**

```bash
# Sirf @smoke tests run karo
npx playwright test --grep @smoke

# Sirf @regression tests run karo
npx playwright test --grep @regression

# @smoke aur @regression dono run karo
npx playwright test --grep "@smoke|@regression"

# @smoke ko SKIP karo, baaki sab run karo
npx playwright test --grep-invert @smoke
```

---

### Example — Mixed Tags (Group + Individual Level)

```javascript
import { test, expect } from "@playwright/test";

test.describe("Checkout Tests", { tag: "@checkout" }, () => {
  // Yeh test @checkout aur @smoke dono hain
  test(
    "TC001 - Add to cart",
    { tag: "@smoke" },
    async ({ page }) => {
      console.log("▶ [checkout][smoke] TC001 running");
      await page.goto("https://example.com/products");
      await page.click("button:has-text('Add to Cart')");
      await expect(page.locator(".cart-count")).toHaveText("1");
    }
  );

  // Yeh test sirf @checkout hai (smoke nahi)
  test("TC002 - Apply coupon code", async ({ page }) => {
    console.log("▶ [checkout] TC002 running");
    await page.goto("https://example.com/cart");
    await page.fill('input[name="coupon"]', "SAVE10");
    await page.click("button:has-text('Apply')");
    await expect(page.locator(".discount")).toBeVisible();
  });

  // Yeh test @checkout aur @regression dono hain
  test(
    "TC003 - Invalid coupon shows error",
    { tag: "@regression" },
    async ({ page }) => {
      console.log("▶ [checkout][regression] TC003 running");
      await page.goto("https://example.com/cart");
      await page.fill('input[name="coupon"]', "FAKECODE");
      await page.click("button:has-text('Apply')");
      await expect(page.locator(".error")).toHaveText("Invalid coupon");
    }
  );
});
```

**Run options:**

```bash
# Sirf checkout smoke tests
npx playwright test --grep "@checkout" --grep "@smoke"

# Checkout ke andar regression tests
npx playwright test --grep "@checkout"
npx playwright test --grep "@regression"

# TC001 aur TC003 dono run hoge (smoke ya regression)
npx playwright test --grep "@smoke|@regression"
```

## Summary

| Feature | Syntax | Scope | Purpose |
|---------|--------|-------|---------|
| `test.use()` | Inside `describe` | Group only | Viewport, locale, colorScheme, etc. set karo |
| `test.use()` | Top-level | Whole file | File ke sab tests ke liye settings |
| `test.describe.configure()` | Inside `describe` | Group only | Serial/parallel mode, retries, timeout |
| Tags on group | `describe("name", { tag: "@x" }, () => {})` | All tests in group | Group ko label karo |
| Tags on test | `test("name", { tag: "@x" }, async () => {})` | Single test | Individual test ko label karo |
| Run by tag | `--grep @tagname` | CLI filter | Sirf tagged tests run karo |
| Skip by tag | `--grep-invert @tagname` | CLI filter | Tagged tests skip karo |
