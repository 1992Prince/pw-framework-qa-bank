# Data Driven Testing — Interview Questions & Answers

---

**Q1. How do you do Data Driven Testing with an array of data in Playwright?**

**Answer:**  
This is the simplest approach. We create an array of objects directly inside our spec file — each object is one test data record with fields like id, name, email, password, etc. Then we wrap our test inside a `for...of` loop — the loop iterates through the array and creates a separate independent test for each record.

For example, if we have 3 records in the array, the loop runs 3 times and Playwright creates 3 separate tests — Test TC001, Test TC002, Test TC003. Each test picks its own data from the current record using `data.email`, `data.password`, etc. We make sure each test name is unique by including the record's ID in the test name — otherwise Playwright throws an error.

This approach is good for quick, small tests — but for real projects we don't use it because the data is hardcoded inside the test file. If data changes, we have to modify the spec file itself. So in real frameworks, we keep data externally in JSON or CSV files.

```typescript
const testData = [
  { id: "TC001", name: "Sarika", email: "sarika@example.com", password: "pass123" },
  { id: "TC002", name: "John",   email: "john@example.com",   password: "pass456" }
];

for (const data of testData) {
  test(`Test ${data.id}`, async ({ page }) => {
    await page.getByTestId("form-email").fill(data.email);
    await page.getByTestId("form-password").fill(data.password);
  });
}
```

**Q2. How do you do Data Driven Testing with a JSON file in Playwright?**

**Answer:**  
We keep our test data in a separate JSON file — like `testdata/registration.json` — which contains an array of objects, each object being one test data record.

In our spec file, we import the JSON file directly using a simple import statement — `import testdata from "../testdata/registration.json"`. JavaScript natively supports JSON, so no extra library is needed. After import, `testdata` is a regular JavaScript array of objects. Then we use the same `for...of` loop to iterate through the array and create a separate test for each record.

The biggest advantage here is separation of data and test logic. If data changes — like adding a new test case or updating an email — we just edit the JSON file. No need to touch the spec file. And if test logic changes, we update the spec file without worrying about data.

One important point — when we import the JSON file, JavaScript creates an in-memory copy. Any modification to `testdata` inside the spec does not change the original JSON file on disk.

```typescript
import testdata from "../testdata/registration.json";

for (const data of testdata) {
  test(`Test ${data.id}`, async ({ page }) => {
    await page.getByTestId("form-email").fill(data.email);
    await page.getByTestId("form-password").fill(data.password);
  });
}
```

**Q3. How do you do Data Driven Testing with a CSV file in Playwright?**

**Answer:**  
When test data comes from business teams or QA leads, it usually comes in CSV or Excel format because that's what non-technical people work with. Unlike JSON, JavaScript cannot import CSV files directly — we need two things: Node.js's built-in `fs` module to read the file from disk, and a parsing library called PapaParse to convert the raw CSV text into JavaScript objects.

So we create a helper function — inside it, first we use `fs.readFileSync()` to read the CSV file as a raw text string. Then we pass that string to `Papa.parse()` with two options — `header: true` so the first row becomes object keys (column names), and `skipEmptyLines: true` to ignore blank rows. PapaParse returns a result object — we take only `.data` from it, which is an array of objects — exactly the same structure as JSON.

After that, the code is identical to the JSON approach — we store the result in a `testdata` variable and use the same `for...of` loop to create tests.

The key takeaway is — whether we use arrays, JSON, or CSV, after loading the data we always end up with the same structure: an array of objects. Only the loading/reading part is different. The test logic stays exactly the same.

```typescript
import Papa from "papaparse";
import fs from "fs";

function readCSV(filePath) {
  const csvData = fs.readFileSync(filePath, "utf-8");
  const parsedData = Papa.parse(csvData, { header: true, skipEmptyLines: true });
  return parsedData.data;
}

const testdata = readCSV("testdata/registration.csv");

for (const data of testdata) {
  test(`Test ${data.id}`, async ({ page }) => {
    await page.getByTestId("form-email").fill(data.email);
    await page.getByTestId("form-password").fill(data.password);
  });
}
```
