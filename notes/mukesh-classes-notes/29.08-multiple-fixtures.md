# Playwright — Multiple Fixtures in One File

---

## Ek Fixture File Mein Multiple Fixtures Kyun?

Aksar ek test ko ek se zyada cheezein chahiye hoti hain — jaise:
- Browser login (UI test ke liye)
- API auth token (API calls ke liye)
- Database connection (data setup ke liye)

Har cheez ke liye alag alag fixture file banana possible hai lekin ek hi file mein multiple fixtures rakhna zyada clean hota hai — ek import se sab mil jaata hai.

---

## Steps — Multiple Fixtures Kaise Banate Hain

### Step 1 — Ek Naya Fixture File Banao

```
fixtures/
  combined.fixture.js   ← naya file
  auth.fixture.js       ← pehle se tha
```

---

### Step 2 — `base` Import Karo

```javascript
import { test as base } from "@playwright/test";
```

Sirf ek baar import karna hai — chahe kitne bhi fixtures banao.

---

### Step 3 — `base.extend({})` Mein Multiple Key-Value Pairs Likho

`base.extend()` ek JavaScript object leta hai. Har key ek fixture ka naam hai, har value ek async function hai. Multiple fixtures matlab multiple key-value pairs — comma se separate:

```javascript
export const test = base.extend({

  fixtureOne: async ({ page }, use) => {
    // setup
    await use(page);
    // teardown
  },

  fixtureTwo: async ({ request }, use) => {
    // setup
    await use(someValue);
    // teardown
  },

  fixtureThree: async ({}, use) => {
    // agar koi built-in fixture nahi chahiye toh empty {} do
    const data = "some value";
    await use(data);
  },

});
```

**Key Rules:**
- Har fixture ke baad **comma** lagao (last ke baad optional)
- Jo built-in fixture chahiye woh pehle parameter `{}` mein maango — `{ page }`, `{ request }`, `{ browser }` etc.
- `use()` mein jo value pass karo — woh test mein fixture ke naam se milegi
- `use()` ke pehle = setup, `use()` ke baad = teardown

---

### Step 4 — Fixture 1: `loggedInUser` — UI Login

```javascript
loggedInUser: async ({ page }, use) => {

  // SETUP: login karo
  await page.goto("https://gauravkhurana.com/practise-api/ui/index.html#/practice");
  await page.getByTestId("form-email").fill("gaurav");
  await page.getByTestId("form-password").fill("gaurav");
  await page.getByTestId("form-submit").click();
  console.log("✅ [loggedInUser] Logged in");

  // HAND OVER: logged-in page test ko do
  await use(page);

  // TEARDOWN: logout karo
  await page.getByTestId("form-reset").click();
  console.log("🔴 [loggedInUser] Logged out");
},
```

**Important points:**
- `{ page }` inject kiya — UI browser page chahiye login ke liye
- `await use(page)` — test ko logged-in page milega
- Test mein `{ loggedInUser }` likhoge — woh yahi `page` hoga

---

### Step 5 — Fixture 2: `apiToken` — API Auth Token

```javascript
apiToken: async ({ request }, use) => {

  // SETUP: API se token fetch karo
  const response = await request.post("https://api.example.com/auth/login", {
    data: {
      username: "testuser",
      password: "testpass123",
    },
  });
  const body = await response.json();
  const token = body.token;
  console.log("✅ [apiToken] Token fetched");

  // HAND OVER: token string test ko do
  await use(token);

  // TEARDOWN: token revoke karo (optional)
  // await request.post("https://api.example.com/auth/logout", {
  //   headers: { Authorization: `Bearer ${token}` },
  // });
  console.log("🔴 [apiToken] Token teardown done");
},
```

**Important points:**
- `{ request }` inject kiya — browser ki zaroorat nahi, sirf API call chahiye
- `await use(token)` — test ko token string milegi (page nahi)
- Test mein `{ apiToken }` likhoge — woh yahi token string hogi

**`loggedInUser` vs `apiToken` ka fark:**

| | `loggedInUser` | `apiToken` |
|-|----------------|------------|
| Inject kiya | `{ page }` | `{ request }` |
| `use()` mein pass kiya | `page` (object) | `token` (string) |
| Test mein milta hai | Browser page | JWT/auth token string |
| Use case | UI test — navigate, click, fill | API test — headers mein token lagao |

---

### Step 6 — `expect` Re-export Karo

```javascript
export { expect } from "@playwright/test";
```

Isse spec files ko alag se `@playwright/test` se import nahi karna padta.

---

### Step 7 — Spec File Mein Import Karo

Import sirf fixture file se karo — `@playwright/test` se nahi:

```javascript
import { test, expect } from "../../fixtures/combined.fixture.js";
```

---

### Step 8 — Tests Likhte Waqt Fixtures Use Karo

**Sirf ek fixture use karo:**

```javascript
test("TC001 - UI test", async ({ loggedInUser }) => {
  await expect(loggedInUser).toHaveURL(/practice/);
});

test("TC002 - API test", async ({ apiToken }) => {
  console.log("Token:", apiToken);
  expect(typeof apiToken).toBe("string");
});
```

**Dono fixtures ek saath use karo:**

```javascript
test("TC003 - UI + API test", async ({ loggedInUser, apiToken }) => {
  // loggedInUser = logged-in browser page
  // apiToken     = auth token string
  await expect(loggedInUser).toHaveURL(/practice/);
  expect(typeof apiToken).toBe("string");
});
```

Ek test mein jo fixture `{}` ke andar likho wahi initialize hota hai — baki initialize hi nahi hote. TC001 mein `apiToken` initialize nahi hoga.

## Complete File Structure

```
fixtures/
  combined.fixture.js     ← loggedInUser + apiToken dono yahan

tests/
  29-hooks-fixtures/
    04-multiple-fixtures.spec.js  ← combined.fixture.js import karke use karta hai
```

---

## Execution Flow — Dono Fixtures Saath (TC003)

```
TC003 starts
  │
  ├── loggedInUser fixture SETUP
  │     goto login page
  │     fill email, password
  │     click submit
  │
  ├── apiToken fixture SETUP
  │     POST /auth/login
  │     token = response.token
  │
  ├── TEST BODY RUNS  ◄─── await use() dono pe call hua
  │     expect(loggedInUser).toHaveURL(...)
  │     expect(typeof apiToken).toBe("string")
  │
  ├── apiToken fixture TEARDOWN
  │     (token revoke — optional)
  │
  └── loggedInUser fixture TEARDOWN
        click logout
```

Fixtures setup reverse order mein tear down hote hain — last defined fixture pehle teardown hota hai.

---

## Quick Reference

| Cheez | Kahan |
|-------|-------|
| Multiple fixtures define karo | `base.extend({ fixture1: ..., fixture2: ... })` |
| Built-in inject karo | Pehle parameter: `{ page }`, `{ request }` |
| Value pass karo test ko | `await use(value)` |
| Test mein use karo | `async ({ fixture1, fixture2 }) => {}` |
| Import karo spec mein | `import { test, expect } from "../../fixtures/combined.fixture.js"` |
